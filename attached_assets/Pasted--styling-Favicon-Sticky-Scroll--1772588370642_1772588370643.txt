Στοιχεία SEO (Για τη Google)
Σύμφωνα με τον Οδηγό SEO της Google, τα στάνταρτ στοιχεία χωρίζονται στις εξής κατηγορίες:
1. Περιεχόμενο και On-Page SEO
Περιεχόμενο "People-first": Δημιουργία αξιόπιστου περιεχομένου που απαντά στις ανάγκες των χρηστών και όχι μόνο στις μηχανές αναζήτησης.
Title Tags & Meta Descriptions: Κάθε σελίδα πρέπει να έχει μοναδικό και περιγραφικό τίτλο (Title Tag) και μια σύντομη περίληψη (Meta Description) που προσελκύει το κλικ.
Σωστή Δομή Επικεφαλίδων: Χρήση ιεραρχικών tags (H1 για τον κύριο τίτλο, H2-H3 για υπότιτλους) για την οργάνωση του κειμένου.
Βελτιστοποίηση Εικόνων: Χρήση περιγραφικού εναλλακτικού κειμένου (Alt Text) και συμπίεση αρχείων για γρήγορη φόρτωση. 
2. Τεχνικά Στοιχεία (Technical SEO)
Ταχύτητα Φόρτωσης & Core Web Vitals: Η Google δίνει μεγάλη βάση στην ταχύτητα και τη σταθερότητα της σελίδας κατά τη φόρτωση.
Mobile-First Indexing: Η ιστοσελίδα πρέπει να είναι responsive, καθώς η Google χρησιμοποιεί κυρίως την έκδοση για κινητά για την αρχειοθέτηση και κατάταξη.
Sitemap & Robots.txt: Ύπαρξη αρχείου XML sitemap για τη διευκόλυνση του crawling και σωστή παραμετροποίηση του robots.txt. 
Σελίδα σφάλματος 404 (custom, branded)
Sitemap Page (HTML) για χρήστες
Breadcrumb navigation σε όλο το site με link κάθε κατηγορία.
3. Δομή και Εμπειρία Χρήστη
Φιλικά προς το SEO URLs: Τα URLs πρέπει να είναι σύντομα, περιγραφικά και να περιέχουν λέξεις-κλειδιά (π.χ. domain.gr/seo-tips/ αντί για domain.gr/?p=123).
Internal Linking: Σωστή διασύνδεση των εσωτερικών σελίδων για να βοηθήσετε τη Google να κατανοήσει τη δομή του site σας.
Schema Markup (Δομημένα Δεδομένα): Χρήση κώδικα schema για την εμφάνιση πλούσιων αποτελεσμάτων (rich snippets) όπως αστέρια αξιολόγησης ή συνταγές. 
4. Εγκυρότητα και Σήματα E-E-A-T
Αυτά βοηθούν τη σελίδα σας να εμφανίζεται στα αποτελέσματα αναζήτησης:
	•	Title Tags & Meta Descriptions: Μοναδικοί τίτλοι και περιγραφές για κάθε σελίδα.
	•	Επικεφαλίδες (H1, H2, H3): Σωστή ιεραρχία κειμένου. Η H1 είναι ο τίτλος, οι H2 είναι οι υπότιτλοι.
	•	Alt Text στις εικόνες: Περιγραφή των εικόνων ώστε να τις "διαβάζει" η Google.
5. Εμπειρία Χρήστη (User Experience - UX)
	•	Mobile Responsiveness: Το site πρέπει να παίζει τέλεια σε κινητά και tablets.
	•	Ταχύτητα Φόρτωσης: Κανείς δεν περιμένει πάνω από 2-3 δευτερόλεπτα.
	•	Καθαρό Menu (Navigation): Εύκολη πλοήγηση για να βρίσκει ο χρήστης αυτό που θέλει γρήγορα.
	•	Readable Fonts: Καθαρά γράμματα και σωστή αντίθεση χρωμάτων.

Interactive Map: Ενσωμάτωση (Embed) χάρτη Google Maps.
Contact Form: Μια καθαρή φόρμα επικοινωνίας με πεδία για Όνομα, Email, Θέμα και Μήνυμα + "Θέλω να προσθέσουμε ένα Consent Checkbox στη φόρμα επικοινωνίας. Η υλοποίηση πρέπει να ακολουθεί το μοντέλο Clickwrap Agreement:
Θέση: Πριν το submit button.

UX: Το checkbox να είναι προεπιλεγμένα κενό (not pre-checked) για συμμόρφωση με τον GDPR.

1. Τοποθέτηση:

2. Τεχνική Υλοποίηση:
API Integration: Το widget πρέπει να τραβάει αυτόματα τις κριτικές από το Google Business Profile (μέσω Google Places API ή κάποιου αξιόπιστου plugin/tool).

3. SEO & Schema:
Aggregate Rating Schema: Παρακαλώ να συμπεριληφθεί το αντίστοιχο Schema Markup, ώστε τα «αστέρια» της βαθμολογίας μας να εμφανίζονται και στα αποτελέσματα αναζήτησης της Google (Rich Snippets).

Το layout να είναι καθαρό και responsive (π.χ. carousel ή grid) ώστε να μην επιβαρύνει την ταχύτητα φόρτωσης της σελίδας.

Οδηγία για Lazy Loading (SEO Friendly)
ΟΧΙ Lazy Load στο LCP (Largest Contentful Paint): Η κύρια εικόνα στην κορυφή της σελίδας (banner/hero image) και το λογότυπο δεν πρέπει να έχουν lazy loading. Πρέπει να φορτώνουν αμέσως.
ΝΑΙ Lazy Load σε όλα τα υπόλοιπα:
Στα 6 blocks (εικόνες).
Στα 3 άρθρα των "Νέων" στην αρχική.
Στο Google Maps και στο Google Reviews Widget (πολύ σημαντικό, γιατί αυτά είναι "βαριά").
Στα QR Codes στη σελίδα επικοινωνίας.
"Θέλω να εφαρμοστεί Native Browser Lazy Loading (loading="lazy") σε όλες τις εικόνες και τα iframes, εκτός από το Hero Section (το πρώτο visual μέρος) κάθε σελίδας, για να πετύχουμε το μέγιστο δυνατό σκορ στα Core Web Vitals της Google."

"Παρακαλώ όπως συμπεριληφθούν στην παράδοση: σωστή διαχείριση 301 redirects, υλοποίηση Breadcrumbs, δημιουργία XML Sitemap/Robots.txt, χρήση Canonical Tags και σύνδεση με Google Search Console/GA4."

"Θέλω το site να περνάει με πράσινο σκορ (90+) τα Google Core Web Vitals και να μην έχει Cumulative Layout Shift (CLS). Αυτό σημαίνει ότι καθώς φορτώνει η σελίδα (π.χ. η μπάρα προόδου ή τα Google Reviews), τα υπόλοιπα στοιχεία δεν πρέπει να 'πηδάνε' και να αλλάζουν θέση."
